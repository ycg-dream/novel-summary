# 经验评估清单（主 Agent 用）

> 每次处理完一本小说后，主 Agent 主动对所有"经验"做一次评估：
> - **A 类（跨书复用）** → 自动追加到 `_learned_patterns.md`
> - **B 类（单书特定）** → 留在项目目录的 `_经验总结_<书名>.md`

## 评估流程

主 Agent 完成全书合并 + 验证 + 清理后，对每条经验问以下问题：

| 问题 | A 类答案 | B 类答案 |
|------|----------|----------|
| 1. 这条经验是否依赖具体书名/人物名？ | ❌ 不依赖 | ✅ 依赖 |
| 2. 这条经验下次处理**任何**小说时都能用？ | ✅ 都能用 | ❌ 仅本书 |
| 3. 这条经验是工具性/流程性/方法论的？ | ✅ 是 | ❌ 是事实记录 |
| 4. 这条经验是否能从具体案例抽象出通用规则？ | ✅ 能 | ❌ 仍是个案 |

**判定标准**：
- 4 题中 ≥ 3 题答 A 类 → A 类（跨书复用）
- 否则 → B 类（单书特定）

## 评估清单（主 Agent 必查 9 项）

完成全书摘要后，主 Agent 主动对照本清单做经验评估：

### 1. R18 策略选择
- [ ] 本书用了哪档策略（保守/中庸/升级）？
- [ ] 该策略命中率如何？
- [ ] 是否有"该策略对本类型小说特别适用"的经验？
- → 如果 A 类，追加到 `_learned_patterns.md` 主题 2

### 2. API 拦截兜底
- [ ] 本书是否有 part 触发 `output new_sensitive (1027)`？
- [ ] 试了几种策略才放弃？
- [ ] 是否有"某种新策略有效/无效"的经验？
- → 如果 A 类，追加到 `_learned_patterns.md` 主题 1

### 3. 跨 part 命名一致性
- [ ] 是否发现命名不一致（如错别字/别字）？
- [ ] 是否在派子 Agent 提示词中注入了命名规范？
- [ ] 是否在合并后做了验证？
- → 如果 A 类，追加到 `_learned_patterns.md` 主题 3

### 4. 合并后验证
- [ ] 合并时是否做了连贯性/排版/命名验证？
- [ ] 发现了什么 bug？
- [ ] 是否有"必须跑 X 验证脚本"的教训？
- → 如果 A 类，追加到 `_learned_patterns.md` 主题 4

### 5. 清理 part 临时文件
- [ ] 合并后是否清理了所有 part 临时 txt 文件？
- [ ] 是否清理了拆分/重试临时文件？
- [ ] 是否有"忘记清理 X 文件"的教训？
- → 如果 A 类，追加到 `_learned_patterns.md` 主题 5

### 6. 子 Agent 提示词模板
- [ ] 派发时是否注入了命名规范？
- [ ] 是否注入了已知上下文？
- [ ] 是否有"提示词必须包含 X"的教训？
- → 如果 A 类，追加到 `_learned_patterns.md` 主题 6

### 7. 进度笔记
- [ ] 是否维护了进度笔记？
- [ ] 进度笔记是否帮上忙（跨会话接续）？
- → 如果 A 类，追加到 `_learned_patterns.md` 主题 7

### 8. 派发节奏
- [ ] 每批派了多少个 part？
- [ ] 5 小时额度下，单个 part 消耗多少？
- [ ] 是否有"应改为每批 X 个"的建议？
- → 如果 A 类，追加到 `_learned_patterns.md` 主题 8

### 9. 开放式结局/末卷处理
- [ ] 末卷是否有特殊结构（开放式结局 / 双线并行 / 大反转）？
- [ ] 是否有"末卷摘要必须包含 X"的教训？
- → 如果 A 类，追加到 `_learned_patterns.md` 主题 9

## 评估输出格式

主 Agent 在对话中输出：

```
=== 经验评估完成 ===

A 类（已追加到 _learned_patterns.md）：
- 主题 X：...

B 类（留在项目目录 _经验总结_<书名>.md）：
- 具体事件 X：...
```

## 完整工作流（更新版）

```
[0] 文本诊断
    ↓
[1] 智能体读小说头尾判断类型
    ↓
[2] Python 切分脚本
    ↓
[3] 子 Agent 派发
    ↓
[4] 子 Agent 读原文 → 输出结构化摘要
    ↓
[5] 主 Agent 质量校验
    ↓
[6] 合并所有 part 摘要 → 全书汇总文件
    ↓
[7] 清理 part 临时文件
    ↓
[8] 维护进度笔记
    ↓
[9] **经验评估 + 沉淀**（本次新增）
    ├─ 读 _learned_patterns.md
    ├─ 对照评估清单 9 项
    ├─ A 类经验自动追加
    └─ B 类经验写入项目目录
```

## 实际操作示例（《烈火凤凰》项目）

主 Agent 完成全书后应该输出：

```
=== 经验评估完成 ===

A 类（已追加到 _learned_patterns.md）：
- 主题 1：API 拦截兜底（part 22 5 种策略全失败）
- 主题 2：R18 策略三档选择（升级版 7/7 part 通过）
- 主题 3：跨 part 命名一致性（白无暇/白无瑕 错别字）
- 主题 4：合并后必须验证（不能简单贴在一起）
- 主题 5：清理 part 临时文件（不能忘记）
- 主题 6：子 Agent 提示词注入命名规范
- 主题 7：进度笔记跨会话接续
- 主题 8：派发节奏（每批 3-7 个 part）
- 主题 9：开放式结局处理

B 类（留在 烈火凤凰_经验总结.md）：
- 《烈火凤凰》part 22 因 R18 密度过高被跳过
- 旧版 part 1-3 用敏感词硬删（保守版）质量差
- 新版 part 4-21 用中庸版质量好
- 测试版 part 17-19 用升级版质量最好
- 末卷 part 26 止于"64章完"（开放式结局）
```
