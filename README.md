# novel-summary

> **把任意大小的中文长篇小说（30 万 ~ 1000 万字）转成结构化摘要的工作流。**
> 切分 → 并行 LLM 摘要 → 合并,主 Agent 只维护进度笔记与质量校验。

支持网文、轻小说、出版小说;**内置 R18 加料章智能标记**(用 ❤ 标记不复述)。

📖 真实战绩:537 万字《放开那个女巫》1498 章 → 27 段并行处理 → 6 小时出全书结构化摘要 → [见案例](SKILL_CASE_537W.md)

---

## ✨ 摘要长啥样?先看一段

处理一本 537 万字的穿越网文,单个 part(约 20 万字)产出如下结构化摘要:

```markdown
【part_007_summary】(约 4.2K 字 / 第 343-397 章)

## 一、核心主线
  主角罗格在北境联合灰堡王国对抗教会与魔兽,
  通过工业化建设(蒸汽机/火枪/印刷术)逐步壮大势力,
  本段完成"第二次北境会战"并启动"新航道开拓"。

## 二、关键剧情节点
  1. **第二次北境会战**(345-360 章):
     教会神罚军突袭灰堡,罗格用"火枪阵+堑壕"战术全歼,
     俘虏红衣主教卡密拉。
  2. **罗兰称王**(378 章):
     原国王病逝,罗格拒绝王位,推举罗兰即灰堡王位。
  3. **新航道首航**(390-395 章):
     蒸汽战舰"曙光号"试航失败,暴露锅炉技术缺陷。

## 三、新增人物
  - **卡密拉**:教会红衣主教,神罚军统帅,被俘后软禁;
    后期为罗格提供教会内部情报。
  - **安德莉亚·范·贝琳**:罗兰未婚妻,擅长外交斡旋。

## 四、世界观/设定扩充
  1. 神罚军真相:并非神授,而是教会用"梦魇水晶"改造的半兽人部队。
  2. 蒸汽机原理:利用煤矿热能推动活塞,首次实战用于战舰。

## 五、政治格局变化
  1. 教会失去对北境 3 个主教区的控制。
  2. 灰堡王国内部"主战派"(罗格)与"稳健派"(罗兰)开始博弈。

## 六、伏笔/未解线索
  1. 罗兰称王后脖颈出现"恶魔之眼"纹路,来源不明。
  2. 神罚军残部退入迷雾山脉,似有新的领袖。
```

**一个 part ≈ 2000-5000 字,涵盖 6 大维度。** 整本 537 万字 → 27 份 part 摘要 + 1 份全书汇总,合并后约 1 MB Markdown。

## 🎯 适合谁 / 不适合谁

### ✅ 适合

- **用 Claude Code / Cursor / VS Code Copilot** 等支持 Claude Skills 的 AI 客户端
- 手里有 **30 万 ~ 1000 万字** 的中文长篇 txt(网文、轻小说、出版小说均可)
- 想 **快速回顾剧情** 但不想重读几百万字
- 需要为长文本训练/研究提供 **结构化摘要**
- 一次想跑 **多本小说**(子 Agent 并行,互不干扰)
- 网文里有 **"加料章"/"车"等 R18 内容**,需要智能标记但不复述

### ❌ 不适合

- **短篇(< 30 万字)**:用一次普通 LLM 调用更快,不需要切分
- **英文/日文/小语种小说**:本 skill 的提示词和章节匹配针对中文优化
- **PDF/EPUB/图片型**:目前只支持纯文本 `.txt`
- **学术论文/技术文档**:走其他摘要方案
- **对"质量 100% 准确"有强需求**:LLM 摘要本质是概率生成,需要人工校对

## 依赖

- **Python 3.8+**(脚本仅用标准库,无第三方依赖)
- **Claude Code / Cursor / VS Code Copilot**(任一支持 Claude Skills 的客户端)
- 子 Agent 推荐模型: **Claude Sonnet 4.6** 或更高(200K 上下文,支持中文长文本)
- 磁盘:每 20 万字切分后约 2-3 MB,合并后单本小说 < 5 MB

## 安装

### 方式 1:作为 Claude Code Skill(推荐)

```bash
# 克隆仓库
git clone https://github.com/ycg-dream/novel-summary.git
# 软链到 skills 目录
ln -s "$(pwd)/novel-summary" ~/.claude/skills/novel-summary
```

### 方式 2:作为 Python 工具包

```bash
git clone https://github.com/ycg-dream/novel-summary.git
cd novel-summary
# 脚本直接用,无需安装
python diagnose_novel.py --path /path/to/novel.txt
```

## 🔄 5 步工作流

```
📖 你的小说.txt (537 万字)
        │
        ▼
① 诊断  ──→  python diagnose_novel.py  (检测编码/谐音字/缺失章节)
        │
        ▼
② 切分  ──→  python split_novel2.py    (切成 N 段 × ~20 万字)
        │
        ▼
③ 并行摘要  ──→  子 Agent × N 独立读 全文 → 写 part_NNN_summary.md
        │              ↑
        │         主 Agent 维护 progress.md 共享上下文
        ▼
④ 合并  ──→  python merge_witch_summaries.py  →  全书摘要汇总.md
        │
        ▼
⑤ 清理  ──→  python cleanup_parts.py  (释放 ~16MB 临时文件)
```

## 🚀 5 分钟上手

1. **调用 Skill**:对 Claude Code 说"用 novel-summary 这个 Skill 给我读《xxx.txt》"
2. **Skill 自动触发**:Claude 会读 `SKILL.md` 并按工作流执行
3. **提供输入**:小说 txt 路径
4. **跑诊断(必做)**:
   ```bash
   python diagnose_novel.py --path /path/to/novel.txt
   ```
   把诊断报告(特别是缺失章节/谐音字/屏蔽词)注入子 Agent 提示词
5. **等待完成**:5-6 个会话 × 30-60 分钟(按 537 万字规模)
6. **清理临时文件(必做)**:
   ```bash
   python cleanup_parts.py
   # 确认删除 (y/N): y
   ```
7. **拿到产物**:单 part 摘要 + 全书汇总

> ⚠️ **不要忘记步骤 4-6**——诊断能避免摘要被"草"="操"等谐音字污染;清理能释放 16MB+ 磁盘。

## 解决的问题

| 痛点 | 传统做法 | Skill 做法 |
|------|----------|-----------|
| 537 万字读不完 | 主 Agent 串行读 → 上下文爆 | 切 27 段 + 子 Agent 并行 |
| 摘要太水 | 抽取式(jieba) | 生成式(子 Agent 独立读全文) |
| 风格不一致 | 主 Agent 状态波动 | 6 大模块结构化提示词 |
| 加料章处理 | 一刀切 | 提示词明确"标记 ❤ 不复述内容" |
| 跨 part 上下文 | 复制前文摘要 → 提示词爆 | 维护 progress.md 共享已知上下文 |

## 关键参数速查

| 参数 | 推荐值 | 备注 |
|------|--------|------|
| 切分粒度 | **20 万字/段** | 50-65 章/段 |
| 子 Agent 模型 | claude-sonnet-4-6 | 200K 上下文 |
| 摘要字数 | 2000-3000 中文字 | 子 Agent 常写到 4000-5000(接受) |
| 提示词长度 | < 5K token | 避免提示词爆炸 |
| 主 Agent 进度笔记 | < 5K token | 共享已知上下文 |
| 加料章处理 | ❤ 标记,不复述 | 提示词明确强调 |

## 排错快速索引

| 症状 | 原因 | 修复 |
|------|------|------|
| 编码检测说"UTF-8"但实际是 GBK | 头部 4KB 是 ASCII 标题 | 用 multi-offset 检测(已内置) |
| **子 Agent 把"草"=操、"曰"=日 一律解读** | 没看到诊断报告 | **跑 diagnose_novel.py + 把谐音字列表注入提示词** |
| 子 Agent 写"前情回顾" | 没看到"不要写前情" | 在提示词首段加 ⚠️ 警告 |
| 章节摘要把广告/书友群也写进来 | 切分前未清理 | 跑 diagnose 识别,再 Python 切掉 |
| 跨 part 人物名不一致 | 子 Agent 不知道前置 | progress.md 同步更新人物名表 |
| 摘要漏掉缺失章节 | 子 Agent 不知道哪些章节缺失 | **把诊断报告的"缺失章节号"注入提示词** |
| 磁盘占用大(十几 GB) | 合并后忘了清理 part 临时文件 | 运行 `python cleanup_parts.py` 删除 |

详见 `SKILL.md` 排错章节。
